### ‚öôÔ∏è How It Works

1.  **Click:** When you click a square (e.g., E2), your browser sends a request to the server at the `/click` endpoint with the square's position.
2.  **Process:** The server determines if this is the start or end of a move, updates the game state, and then performs a **HTTP 302 Redirect** back to this README page.
3.  **Refresh:** The page reloads, forcing GitHub's Markdown renderer to re-fetch all 64 dynamic images from the `/square_img` endpoint. Since the game state has changed, the images served by your backend (including piece placement and potential highlighting) are now updated.

### üíª Development

This project uses Python to manage the chess logic and a web framework (like Flask or FastAPI) to handle the two critical endpoints:

* **`/square_img?game=<ID>&sq=<SQUARE>`:** Dynamically generates the piece image (with background color/highlight) for a single square.
* **`/click?game=<ID>&sq=<SQUARE>`:** Handles the move logic and forces the redirect back to GitHub.
